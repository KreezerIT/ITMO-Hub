# Лабораторная работа №6. *Работа с модулем FUSE*

[Файл с полным ТЗ+Теория](OS_LabDescription_6_hard_1.pdf)

## Задание на лабораторную работу

Вам предстоит написать собственную имплементацию **FUSE**, выполняющую **одно из предложенных заданий**.

Рекомендуется придерживаться следующих шагов:

1. **Выбор языка программирования.** Можно выбрать любой, у большинства популярных ЯП есть библиотеки для работы с FUSE (например, **C/C++**, **Rust** и т.д.).
2. **Настройка инструментов.** Установить необходимые пакеты (например, `fuse`, `libfuse-dev`), настроить IDE или подключение к виртуальной машине.
3. **Написание кода.** Переопределить логику работы необходимых операций (`read`, `write`, `mkdir`, `readlink` и т.д.), решить какие операции изменяются, какие остаются «как есть», а какие будут `noop`.
4. **Тестирование.** Убедиться, что выполнение команд `ls`, `cat`, `rm`, `rmdir`, `ln`, `chown`, `cd`, `echo 1 > file` не приводит к краху файловой системы.

## Варианты заданий

1. **Хранилище файлов в Discord/Telegram**  
   Реализовать работу с **API Discord или Telegram** для сохранения файлов в каналах и отображения их через FUSE-файловую систему.

2. **Конвертация файлов на лету**  
   Реализовать файловую систему, которая при обращении к файлам в одном формате (например, `jpg`) конвертирует и отдаёт данные из исходных файлов (например, `png`) «на лету».

3. **Группировка mp3 файлов**  
   Реализовать группировку файлов формата **mp3** по тегам (**Artist**, **Year**, **Genre**) с построением соответствующей иерархии каталогов в примонтированной ФС (режим **read only**).

4. **Авторазархивирование**  
   Реализовать файловую систему, в которой каждый архив в исходной директории отображается как каталог с содержимым архива (минимум **два формата архивов** на выбор), при этом сами архивные каталоги **read only**.

5. **ФС с кастомными правилами**  
   Реализовать файловую систему, которая в зависимости от **конфигурационного файла** и типа файла выполняет разные операции при чтении/записи и при листинге директорий.

6. **Своё задание**  
   Собственная идея по использованию **FUSE**, согласованная и формализованная (ТЗ) совместно с преподавателем.
