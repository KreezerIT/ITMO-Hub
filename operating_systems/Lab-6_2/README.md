# Лабораторная работа №6 (вариант повышенной сложности). *Экспериментальная оценка параметров производительности операционной системы*

[Файл с полным ТЗ+Теория](OS_LabDescription_6_hard_2.pdf)

## Цель лабораторной работы

Оценить реальные накладные расходы на параллельное выполнение задач в условиях преимущественного использования ресурса процессора и ресурса дисковой подсистемы.

---

## Задание на лабораторную работу

Необходимо организовать и провести **две группы экспериментов** и оформить отчет по их результатам.

---

## Группа 1. Вычислительно сложные задачи (CPU-bound)

1. **Разработать вычислительно сложный алгоритм**, который:
    - получает на вход несколько числовых параметров;
    - на выходе выдает одно или несколько числовых значений;
    - выполняется примерно **2–3 секунды** для одного набора параметров;
    - минимально использует ввод/вывод.

2. **Эксперимент 1 (последательный запуск, 1 процессор):**
    - настроить ВМ на использование **1 процессора**;
    - написать скрипт, который при параметре `N` **последовательно** запускает `N` вычислений для разных входных параметров;
    - вторым скриптом для каждого `N` от 1 до 20 **10 раз** запускать первый через `time` и фиксировать полное время выполнения;
    - для каждой серии из 10 измерений посчитать **среднее время** и построить график зависимости среднего времени от `N`.

3. **Эксперимент 2 (параллельный запуск, 1 процессор):**
    - написать скрипт, который при параметре `N` **параллельно** (без ожидания завершения предыдущих) запускает `N` вычислений для разных входных параметров;
    - повторить шаги измерений и построения графика, аналогичные эксперименту 1.

4. **Эксперименты 3 и 4 (2 процессора):**
    - настроить ВМ на использование **2 процессоров**;
    - повторить эксперименты 1 и 2 (последовательный и параллельный запуск) в тех же режимах;
    - в итоге получить **4 графика** (последовательно/параллельно × 1/2 процессора).

---

## Группа 2. Задачи с большими объемами ввода/вывода (IO-bound)

1. **Разработать алгоритм обработки больших файлов**, который:
    - работает с массивами данных большого объема (например, файлы по ~10 МБ);
    - в каждом файле последовательно:
        - считывает очередное целое число;
        - умножает его на 2;
        - дописывает результат **в конец того же файла**;
    - выполняет **чтение → изменение → запись** для каждого значения *поэлементно*, а не одним большим буфером;
    - обрабатывает один файл за **2–3 секунды** (размер подобрать экспериментально).

2. **Подготовить набор файлов** (например, 20 файлов), каждый из которых используется **своим экземпляром алгоритма**.

3. **Повторить план экспериментов из Группы 1:**
    - провести эксперименты с последовательным и параллельным запуском обработки файлов;
    - выполнить их при конфигурациях с **1** и **2 процессорами**;
    - получить еще **4 графика** (итого 8 по обеим группам).

## Дополнительные исследования (по желанию)

При желании дополнительно оценить:
- потребление памяти отдельными процессами;
- загрузку процессора;
- другие параметры ОС.

Выбор параметров и методов их оценки остается за экспериментатором.

---

## Требования к отчету

В отчете должны быть представлены:

1. **Детали реализации алгоритмов** (для обеих групп экспериментов).
2. **Параметры виртуальной машины и хостовой системы**, на которой проводились эксперименты.
3. **Уточненный план эксперимента**, если он был модифицирован.
4. **Графики** с пояснением, к какому эксперименту относится каждый график.
5. **Выводы** с попыткой теоретически объяснить наблюдаемые на графиках эффекты.
